The development process:

  • Write code locally
  • Commit that code
  • Push that commit to the git repository
  • A build is triggered on Codeship
  • Codeship deploys the build to testing.blackbullion.com
  • We verify our work in test
  • Merge the develop branch into master
  • Another build is triggered on Codeship, which deploys the build to production

Write code locally

  • explain "locally"
  • this is an overview, don't dive too deep into coding practices like TDD

Commit that code

  • git is a piece of software which I'll go into in more depth later
  • committing your work is like creating a new file every time you save something on your computer
  • save files are stored in a repository/repo
  • you're able to look at the state of the project at specific points in time

Push that commit to the git repository

git push origin develop
  • GitHub is a place where git repos can be stored remotely
  • allows teams of developers to all work from the same codebase

A build is triggered on Codeship

  • When we push our work to GitHub, builds are triggered automatically
  • Codeship is a PaaS, talk more about what it does later
  • It's an automated way of ensuring that the code is in a good enough state for the product to be able to run on a server

Codeship deploys the build to testing.blackbullion.com

  • Automated deploys reduce human error
